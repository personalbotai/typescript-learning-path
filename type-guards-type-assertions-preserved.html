Type Guards dan Type Assertions

Ketika bekerja dengan union types, kita seringkali perlu membedakan tipe mana yang sebenarnya dimiliki sebuah nilai. TypeScript menyediakan type guards untuk memeriksa tipe saat runtime, dan type assertions untuk memberitahu compiler tentang tipe yang kita yakin.

1 Typeof Type Guard

Operator typeof bisa digunakan dalam type guard untuk memeriksa tipe primitif (string, number, boolean, symbol, undefined, object, function).

function printValue(value: string | number | boolean) { if (typeof value === "string") { console.log("String:", value.toUpperCase()); } else if (typeof value === "number") { console.log("Number:", value.toFixed(2)); } else { console.log("Boolean:", value); } }

TypeScript akan mempersempit tipe di dalam cabang if berdasarkan hasil type guard.

2 Instanceof Type Guard

Gunakan instanceof untuk memeriksa apakah sebuah objek adalah instance dari kelas atau constructor tertentu.

class Dog { bark() { console.log("Woof!"); } } class Cat { meow() { console.log("Meow!"); } } function makeSound(pet: Dog | Cat) { if (pet instanceof Dog) { pet.bark(); // TypeScript tahu $pet$ bertipe Dog di sini } else { pet.meow(); // TypeScript tahu $pet$ bertipe Cat } }

3 Custom Type Guards

Kita bisa membuat fungsi khusus yang mengembalikan value is Tipe (type predicate) untuk memeriksa properti objek.

interface Fish { swim(): void; } interface Bird { fly(): void; } function isFish(pet: Fish | Bird): pet is Fish { return ("swim" in pet); } function getPet(pet: Fish | Bird) { if (isFish(pet)) { pet.swim(); // OK, pet bertipe Fish } else { pet.fly(); // OK, pet bertipe Bird } }

Fungsi isFish adalah type predicate. TypeScript menggunakan ini untuk memperketat tipe di dalam cabang if.

4 Type Assertions

Terkadang kita tahu lebih baik daripada TypeScript tentang tipe suatu nilai. Gunakan as Tipe atau <Tipe>value untuk assert tipe.

// Contoh: DOM API mengembalikan Element | Text, tapi kita tahu itu adalah Button const button = document.getElementById("myBtn") as HTMLButtonElement; // Sama dengan syntax alternatif: const button2 = <HTMLButtonElement>document.getElementById("myBtn"); // Tapi hati-hati: ini TIDAK memeriksa di runtime! const someValue: any = "hello"; const strLength = (someValue as string).length; // OK untuk compiler, tapi bisa crash jika someValue bukan string

5 Discriminated Unions (Tagged Unions)

Jika setiap anggota union memiliki properti discriminant (biasanya kind atau type) yang nilainya adalah literal string, TypeScript dapat melakukan exhaustive checking.

type Shape = | { kind: "circle"; radius: number; } | { kind: "rectangle"; width: number; height: number; } | { kind: "square"; size: number; }; function getArea(shape: Shape): number { switch (shape.kind) { case "circle": return Math.PI * shape.radius ** 2; case "rectangle": return shape.width * shape.height; case "square": return shape.size ** 2; } }

Jika kita lupa menambahkan case "square", TypeScript akan memberikan error "Exhaustiveness check". Ini membuat kode lebih aman dan mudah di-refactor.
