Function Types di TypeScript

Di TypeScript, function adalah "first-class citizens" - bisa disimpan di variabel, dikembalikan dari function lain, dan diatribusikan tipe dengan ketat. Memahami function types adalah kunci untuk type-safe code.

1. Function Declaration vs Expression

Dua cara mendefinisikan function, masing-masing punya karakteristik type inference yang berbeda.

// Function Declaration (named function)
function tambah(a: number, b: number): number {
    return a + b;
}
// Type: (a: number, b: number) => number

// Function Expression (assign ke variabel)
let kali = function(x: number, y: number): number {
    return x * y;
};
// Type: (x: number, y: number) => number

// Arrow Function Expression
const bagi = (x: number, y: number): number => {
    return x / y;
};
// Type: (x: number, y: number) => number

// Type inference: jika tidak ada explicit return type, TypeScript akan menebak
let tanpaTipe = function(a, b) { return a + b; };
// Error: Parameter 'a' implicitly has an 'any' type (strict mode)
// Harus: let tanpaTipe = (a: number, b: number) => a + b;

2. Type Alias untuk Function

Buat type alias khusus untuk function signature agar reusable.

// Define function type
type BinaryOperation = (a: number, b: number) => number;

// Gunakan type alias
let tambah: BinaryOperation = (a, b) => a + b;
let kali: BinaryOperation = (a, b) => a * b;

// Bisa juga dengan generic
type Comparator = (a: T, b: T) => number; // -1, 0, 1

let numberComparator: Comparator = (a, b) => a - b;
let stringComparator: Comparator = (a, b) => a.localeCompare(b);

// Function type dengan optional dan rest parameters
type ConfigFunction = (debug?: boolean, ...args: string[]) => void;
let config: ConfigFunction = (debug = false, ...args) => {
    if (debug) console.log(args);
};

3. Optional dan Default Parameters

// Optional parameter ( Dengan ?)
function greet(name: string, greeting?: string): string {
    if (greeting) {
        return `${greeting}, ${name}!`;
    }
    return `Hello, ${name}!`;
}
greet("Alice"); // "Hello, Alice!"
greet("Bob", "Hi"); // "Hi, Bob!"

// Default parameter
function power(base: number, exponent: number = 2): number {
    return Math.pow(base, exponent);
}
power(5);    // 25 (exponent default 2)
power(2, 3); // 8

// Kombinasi
function createUser(name: string, age: number = 18, active: boolean = true): User {
    return { name, age, active };
}

4. Rest Parameters

Collect semua arguments menjadi array.

// Rest parameter (...args)
function sumAll(...numbers: number[]): number {
    return numbers.reduce((a, b) => a + b, 0);
}
sumAll(1, 2, 3, 4); // 10

function concatStrings(sep: string, ...strings: string[]): string {
    return strings.join(sep);
}
concatStrings("-", "a", "b", "c"); // "a-b-c"

// Rest harus parameter terakhir
// function badRest(...a: number, b: string) {} // Error!

5. Function Overloads

TypeScript memungkinkan mendefinisikan beberapa signatures untuk satu implementation. Berguna ketika function bisa dipanggil dengan argumen berbeda tipe.

// Overload signatures
function format(input: string): string;
function format(input: number): string;
function format(input: boolean): string;
// Implementation (haruså…¼å®¹ semua signatures)
function format(input: any): string {
    if (typeof input === "string") {
        return `String: ${input.toUpperCase()}`;
    } else if (typeof input === "number") {
        return `Number: ${input.toFixed(2)}`;
    } else if (typeof input === "boolean") {
        return `Boolean: ${input ? "true" : "false"}`;
    }
    return "Unknown";
}

// Pemanggilan
format("hello"); // OK, returns string
format(42);      // OK, returns string
format(true);    // OK, returns string
// format({});    // Error - tidak ada overload untuk object

Aturan: Overload signatures hanya berisi deklarasi (tanpa body). Implementation harus ditulis sekali dengan parameter any atau union type yang mencakup semua overloads.

6. This Parameters

Kadang function menggunakan this. TypeScript butuh bantuan untuk tahu tipe this.

interface Person {
    nama: string;
    greet(this: Person): string; // explicit this type
}

let p: Person = { nama: "Alice", greet: function() { return `Hi, ${this.nama}`; } };
console.log(p.greet()); // "Hi, Alice"

// Arrow function tidak memiliki own this, gunakan lexical this
class Counter {
    count: number = 0;
    increment = () => { this.count++; }; // arrow, this lexical dari class instance
    getCount = (): number => this.count;
}

// Function dengan this parameter bisa dipanggil dengan call/apply
function logThis(this: any) {
    console.log(this);
}
logThis.call({ x: 1 }); // OK, this = {x:1}

Latihan: Build Type-Safe Event Emitter

type EventHandler = (data: T) => void;

class EventEmitter> {
    private listeners: Partial[]>> = {};

    on(event: K, handler: EventHandler): void {
        if (!this.listeners[event]) {
            this.listeners[event] = [];
        }
        this.listeners[event]!.push(handler);
    }

    emit(event: K, data: Events[K]): void {
        const handlers = this.listeners[event];
        if (handlers) {
            for (let handler of handlers) {
                handler(data);
            }
        }
    }
}

// Usage:
interface MyEvents {
    "user:login": { username: string; timestamp: Date };
    "error": { message: string; code: number };
}

let emitter = new EventEmitter();
emitter.on("user:login", (data) => {
    console.log(`${data.username} logged in at ${data.timestamp}`);
});
emitter.emit("user:login", { username: "alice", timestamp: new Date() });
// Type-safe: emit dengan data yang sesuai signature event

Rangkuman

  • Function types: (params) => returnType
  • Bisa buat type alias untuk function signature
  • Optional parameters ?, default parameters, rest parameters ...
  • Function overloads untuk multiple signatures
  • this parameter untuk explicit this typing
  • TypeScript mengecek jumlah dan tipe argumen, serta tipe return
