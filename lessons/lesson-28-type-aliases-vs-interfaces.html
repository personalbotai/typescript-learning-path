Type Aliases vs Interfaces

Di TypeScript, kita bisa mendefinisikan tipe custom menggunakan type (type alias) atau interface. Keduanya mirip, tapi ada beberapa perbedaan penting yang perlu dipahami untuk memilih yang tepat.

1 Interface (Antarmuka)

interface digunakan untuk mendefiniskan bentuk objek (object shape). Dapat di-extend (mewarisi) dan di-implement oleh class.

interface User { name: string; age: number; email?: string; // optional } // Extending interface interface Admin extends User { role: string; } class PersonService { createUser(user: User) { /* ... */ } }

2 Type Alias (Alias Tipe)

type membuat alias untuk tipe apapun, bukan hanya object. Bisa digunakan untuk union, tuple, function types, dll.

// Type alias untuk object type User = { name: string; age: number; email?: string; }; // Bisa juga untuk union, tuple, dll type ID = string | number; type Coordinates = [number, number]; type Callback = (data: string) => void; // Type alias bisa menggunakan union dengan intersection type AdminUser = User & { role: string };

4 Kapan Memilih Interface?

  • Mendefinisikan bentuk objek saja (tanpa union, tuple, dll)
  • Butuh declaration merging (API bisa diperluas di tempat lain)
  • Akan di-implement oleh class
  • Mau menggunakan extends untuk warisan interface (lebih familiar)

5 Kapan Memilih Type Alias?

  • Butuh union, intersection, atau tuple
  • Mendefinisikan tipe function (callback signatures)
  • Membuat mapped types atau conditional types
  • Tipe yang lebih kompleks atau tidak hanya object

6 Contoh Kombinasi

Seringkali kita gunakan keduanyabersamaan. Interface untuk objek utama, type alias untuk union atau tipe lain.

// Interface untuk objek utama interface User { id: number; name: string; } // Type alias untuk union state type UserState = "active" | "inactive" | "banned"; interface ExtendedUser extends User { state: UserState; // menggunakan type alias di dalam interface } const user: ExtendedUser = { id: 1, name: "Alice", state: "active" };
