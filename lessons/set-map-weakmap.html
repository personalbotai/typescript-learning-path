Collection Types di TypeScript

Selain array, TypeScript menyediakan collection types lain yang lebih specialized: Set, Map, dan WeakMap. Mereka memiliki karakteristik unik untuk use-case tertentu.

1. Set

Set adalah koleksi unik nilai. Tidak ada duplikat. Nilai bisa berupa primitive atau object.

// Membuat Set
let numbers = new Set([1, 2, 3, 4, 5, 5, 5]);
console.log(numbers); // Set { 1, 2, 3, 4, 5 } - duplikat otomatis dihapus

// Tambah nilai
numbers.add(6);
numbers.add(2); // tidak ditambahkan karena sudah ada
console.log(numbers.size); // 6

// Cek keberadaan
console.log(numbers.has(3)); // true
console.log(numbers.has(10)); // false

// Hapus nilai
numbers.delete(1);
console.log(numbers.has(1)); // false

// Clear semua
numbers.clear();
console.log(numbers.size); // 0

// Iterasi
let fruits = new Set(["apple", "banana", "cherry"]);
for (let fruit of fruits) {
    console.log(fruit);
}
// apple, banana, cherry (urutan insert)

// Convert ke array
let arr: string[] = Array.from(fruits);
console.log(arr); // ["apple", "banana", "cherry"]

Use case: Gunakan Set ketika Anda butuh koleksi unik tanpa duplikat, misalnya daftar ID, tag, atau username yang sudah diproses.

2. Map

Map adalah koleksi pasangan key-value di mana key bisa berupa tipe apa pun (tidak hanya string/symbol seperti object).

// Membuat Map
let scores = new Map();
scores.set("Alice", 95);
scores.set("Bob", 87);
scores.set("Charlie", 92);

// Ambil nilai
console.log(scores.get("Alice")); // 95
console.log(scores.get("David")); // undefined

// Cek key
console.log(scores.has("Bob")); // true

// Ukuran
console.log(scores.size); // 3

// Hapus
scores.delete("Bob");
console.log(scores.has("Bob")); // false

// Iterasi
for (let [name, score] of scores) {
    console.log(`${name}: ${score}`);
}
// Alice: 95, Charlie: 92

// Keys, Values, Entries
console.log([...scores.keys()]); // ["Alice", "Charlie"]
console.log([...scores.values()]); // [95, 92]
console.log([...scores.entries()]); // [["Alice",95],["Charlie",92]]

// Clear
scores.clear();
Map<K, V>

Key bisa berupa object, number, boolean - apa saja. Perfect untuk dictionary dinamis.

3. WeakMap

WeakMap mirip Map, tetapi key harus berupa object dan tidak mencegah garbage collection. Jika key tidak direferensi lagi, entry akan dihapus otomatis.

// WeakMap: key harus object
let weakMap = new WeakMap();

let obj1 = { id: 1 };
let obj2 = { id: 2 };

weakMap.set(obj1, "data untuk obj1");
weakMap.set(obj2, "data untuk obj2");

console.log(weakMap.get(obj1)); // "data untuk obj1"

// Tidak ada .size, .clear(), .keys(), .values() karena weak
// weakMap.size; // Error!

// Hanya ada: get, set, has, delete

// Jika obj1 tidak direferensi, entry akan dihapus oleh GC
obj1 = null; // sekarang weakMap mungkin sudah kosong untuk key itu

WeakMap use case: Simpan metadata tambahan untuk object tanpa mencegah object tersebut di-GC. Contoh: private data, cache per-instance.

4. Perbandingan: Object, Map, WeakMap

Feature Object Map WeakMap
Key types string | symbol any object only
Key uniqueness string keys === (strict) === (object identity)
Iteration order No guaranteed Insertion order N/A (no iteration)
Size property Manual count .size None
Garbage collection Prevents GC Prevents GC Does NOT prevent GC
Use case Simple dict, JSON General key-value Metadata, private data

5. Contoh Praktis: Cache dengan Map

Gunakan Map untuk membuat cache yang efisien.

// Simple memoization cache
let fibonacciCache = new Map();

function fibonacci(n: number): bigint {
    // Cek cache
    if (fibonacciCache.has(n)) {
        return fibonacciCache.get(n)!; // non-null assertion
    }

    // Hitung
    let result: bigint;
    if (n <= 1) {
        result = BigInt(n);
    } else {
        result = fibonacci(n - 1) + fibonacci(n - 2);
    }

    // Simpan ke cache
    fibonacciCache.set(n, result);
    return result;
}

console.log(fibonacci(50)); // Cached after first call
console.log(`Cache size: ${fibonacciCache.size}`); // 51

Latihan: Remove Duplicates dengan Set

// Fungsi untuk menghapus duplikat dari array
function unique(arr: T[]): T[] {
    return [...new Set(arr)];
}

// Contoh
let numbers = [1, 2, 2, 3, 4, 4, 5];
console.log(unique(numbers)); // [1,2,3,4,5]

let strings = ["a", "b", "a", "c", "b"];
console.log(unique(strings)); // ["a","b","c"]

// Complex objects? hati-hati:
let objs = [{id:1}, {id:1}];
console.log(unique(objs)); // [{id:1}, {id:1}] - berbeda reference!

Kesalahan Umum

WeakMap iteration

let wm = new WeakMap();
wm.forEach((v,k) => console.log(k,v)); // Error! WeakMap tidak support iteration

Object as Map

let obj = {};
obj[1] = "one";   // key di-convert ke string: "1"
obj[[1,2]] = "array"; // "[1,2]" - bukan array reference!
// Gunakan Map untuk non-string keys!

Rangkuman

  • Set: koleksi unik, otomatis hapus duplikat
  • Map: key-value dengan key bisa any type, insertion order preserved
  • WeakMap: Map dengan key object-only, GC-friendly (key bisa di-GC)
  • Gunakan Set untuk deduplication, Map untuk dictionary dinamis, WeakMap untuk metadata
  • WeakMap tidak memiliki size, iteration, atau clear()
