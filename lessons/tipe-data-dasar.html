Tipe Data Dasar (string, number, boolean, null, undefined)

Modul 1: Dasar-Dasar TypeScript

Kemajuan Belajar 0%

Pengantar Tipe Data

TypeScript memiliki sistem tipe yang kuat dan statis. Setiap variabel harus memiliki tipe yang jelas, atau TypeScript akan mencarinya melalui type inference. Mari kita bahas tipe data primitif yang paling sering digunakan.

1. String (Teks)

Tipe string untuk teks, bisa menggunakan tanda kutip ganda, tanda kutip tunggal, atau backtick (template literal).

// Deklarasi string
let nama: string = "Archon";
let pesan: string = 'Hello, World!';
let template: string = `Nama: ${nama}, Umur: 42`;

// Operasi string
console.log(pesan.length);           // 13
console.log(pesan.toUpperCase());    // "HELLO, WORLD!"
console.log(pesan.slice(0, 5));      // "Hello"
string

2. Number (Angka)

Tipe number mencakup integer, float, hex, octal, binary, dan bigint.

// Angka biasa
let usia: number = 42;
let phi: number = 3.14159;
let hex: number = 0xff;      // 255
let binary: number = 0b1010; // 10

// Operasi matematika
let hasil: number = 10 + 5;   // 15
let pembagian: number = 10 / 3; // 3.333...

// BigInt untuk angka sangat besar
let big: bigint = 9007199254740991n;
number bigint

3. Boolean (Logika)

Tipe boolean hanya punya dua nilai: true atau false.

let aktif: boolean = true;
let selesai: boolean = false;

// Operasi logika
let dan: boolean = aktif && selesai;   // false
let atau: boolean = aktif || selesai;  // true
let negasi: boolean = !aktif;          // false

// Perbandingan
let lebihBesar: boolean = 10 > 5;      // true
let sama: boolean = 10 === "10";       // false (strict equality)
boolean

4. Null dan Undefined

null dan undefined adalah tipe khusus yang menandakan "tidak ada nilai".

Null

Null adalah nilai yang sengaja dikelilingi untuk menandakan "tidak ada objek".

let data: string | null = null;
data = "Ada nilai"; // valid

Undefined

Undefined berarti variabel belum diberi nilai.

let belumDiatur: undefined = undefined;
let optional: string | undefined;
null undefined

Type Inference (Inferensi Tipe)

TypeScript bisa menebak tipe berdasarkan nilai yang diberikan. Anda tidak selalu harus menulis tipe secara eksplisit.

// TypeScript menebak tipe otomatis
let pesan = "Hello";      // string
let angka = 42;           // number
let aktif = true;         // boolean
let campur = "text";      // string (tetap string, tidak jadi any)

// Tapi hati-hati:
let arr = [1, 2, 3];     // number[]
arr.push("4");            // Error! Tidak bisa menambahkan string ke number[]

Catatan: Gunakan any hanya jika benar-benar perlu. Hindari karena akan melemahkan type-checking.

Latihan: Tentukan Tipe

Apa tipe dari nilai-nilai berikut? (Jawab tanpa menjalankan kode)

2. 3.14 number
3. false boolean
4. null null

Kesalahan Umum

Type Mismatch

let angka: number = "42"; // Error: Type 'string' is not assignable to type 'number'

Implicit any

function foo(x) { return x; } // Error: Parameter 'x' implicitly has an 'any' type

Rangkuman

  • string untuk teks
  • number untuk angka (termasuk bigint)
  • boolean untuk true/false
  • null dan undefined untuk nilai kosong
  • Type inference membantu mengurangi boilerplate, tapi tambahkan tipe eksplisit untuk clearer code
  • Hindari any jika memungkinkan
