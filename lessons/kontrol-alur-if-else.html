Kontrol Alur: if/else, switch, ternary

Modul 1: Dasar-Dasar TypeScript

Kemajuan Belajar 0%

Apa itu Kontrol Alur?

Kontrol alur memungkinkan kita mengatur eksekusi kode berdasarkan kondisi tertentu. Dengan kontrol alur, program bisa membuat keputusan dan mengambil jalur yang berbeda.

Konsep: TypeScript menggunakan struktur kontrol yang sama seperti JavaScript. TypeScript hanya menambahkan type checking pada ekspresi yang digunakan dalam kondisi.

1. if / else if / else

Struktur if mengeksekusi blok kode jika kondisi benar.

let nilai: number = 85;
let grade: string;

if (nilai >= 90) {
    grade = "A";
} else if (nilai >= 80) {
    grade = "B";
} else if (nilai >= 70) {
    grade = "C";
} else if (nilai >= 60) {
    grade = "D";
} else {
    grade = "E";
}

console.log(`Grade: ${grade}`); // Grade: B

Aturan Penting:

  • Kondisi dalam if harus bertipe boolean
  • Blok else opsional, bisa hanya if saja
  • Bisa chain else if sebanyak needed
  • Hanya satu blok yang dieksekusi (yang kondisi pertama true)

2. Nested if (if bersarang)

If di dalam if untuk kondisi yang lebih kompleks.

let umur: number = 20;
let status: string = "pelajar";

if (umur >= 18) {
    if (status === "pelajar") {
        console.log("Dewara muda pelajar");
    } else {
        console.log("Dewara muda pekerja");
    }
} else {
    console.log("Anak-anak");
}

Output: Dewara muda pelajar

3. switch-case

Cocok untuk memilih salah satu dari banyak kemungkinan nilai suatu variabel.

let hari: number = 3; // 1=Senin, 2=Selasa, dst
let namaHari: string;

switch (hari) {
    case 1:
        namaHari = "Senin";
        break;
    case 2:
        namaHari = "Selasa";
        break;
    case 3:
        namaHari = "Rabu";
        break;
    case 4:
        namaHari = "Kamis";
        break;
    case 5:
        namaHari = "Jumat";
        break;
    case 6:
    case 7:
        namaHari = "Akhir Pekan";
        break;
    default:
        namaHari = "Invalid";
        break;
}

console.log(namaHari); // Rabu

Jangan lupa break! Jika lupa, eksekusi akan "fall through" ke case berikutnya.

4. Ternary Operator (Operator Tersier)

Cara singkat untuk if-else yang mengembalikan nilai.

let umur: number = 18;
let status: string;

// if-else biasa
if (umur >= 18) {
    status = "Dewasa";
} else {
    status = "Anak-anak";
}

// dengan ternary
status = umur >= 18 ? "Dewasa" : "Anak-anak";

console.log(status); // Dewasa

Format: kondisi ? nilaiJikaTrue : nilaiJikaFalse

5. Truthy dan Falsy

Di TypeScript/JavaScript, beberapa nilai dianggap "falsy" (false) saat dikonversi ke boolean:

Falsy Values

  • false
  • 0 dan -0
  • "" (string kosong)
  • null
  • undefined
  • NaN

Truthy Values

Semua nilai selain falsy dianggap truthy, termasuk:

  • "0" (string "0")
  • "false" (string "false")
  • [] (array kosong)
  • {} (object kosong)
  • 42, "hello", dll
let x: any = 0;

if (x) {
    console.log("Truthy");
} else {
    console.log("Falsy"); // Ini akan dieksekusi
}

// Gunakan strict checking untuk menghindari bug
if (x === 0) {
    console.log("x adalah 0");
}

Latihan: Buat Logika Kontrol Alur

Tulis kode untuk menentukan apakah sebuah tahun adalah kabisat:

function isKabisat(tahun: number): boolean {
    // Tahun kabisat jika:
    // - Habis dibagi 4 TAPI
    // - Tidak habis dibagi 100 KECUALI habis dibagi 400
    if (tahun % 400 === 0) {
        return true;
    } else if (tahun % 100 === 0) {
        return false;
    } else if (tahun % 4 === 0) {
        return true;
    } else {
        return false;
    }
}

console.log(isKabisat(2000)); // true (kabisat)
console.log(isKabisat(1900)); // false (bukan kabisat)
console.log(isKabisat(2024)); // true (kabisat)
console.log(isKabisat(2023)); // false (bukan kabisat)

Kesalahan Umum

Assignment vs Comparison

if (x = 5) { } // BUG! Ini assignment, bukan comparison
// Harusnya: if (x === 5) { }

Lupa break di switch

switch (x) {
    case 1:
        console.log("satu");
        // lupa break! akan ke case 2
    case 2:
        console.log("dua"); // akan dieksekusi meski x=1
        break;
}

Rangkuman

  • if/else if/else untuk kondisi berantai
  • switch-case cocok untuk banyak nilai tetap, jangan lupa break
  • kondisi ? a : b (ternary) untuk if-else satu baris
  • Pahami truthy/falsy values untuk menghindari bug
  • Selalu gunakan strict equality === dalam kondisi
  • TypeScript akan memeriksa tipe kondisi harus boolean (atau truthy/falsy yang bisa dikonversi)
