Polymorphism (Banyak Bentuk)

Polymorphism adalah kemampuan sebuah object untuk mengambil banyak bentuk. Dalam OOP, ini berarti subclass dapat meng-override method dari superclass, dan kita bisa memperlakukan berbagai subclass sama sebagai tipe superclass/interface.

Konsep: "Satu interface, banyak implementasi." TypeScript achieve polymorphism melalui inheritance dan interface.

1. Polymorphism dengan Inheritance

Subclass meng-override method parent, dan dipanggil melalui reference parent type.

class Animal {
    constructor(public name: string) {}

    // Method yang akan di-override
    makeSound(): void {
        console.log(`${this.name} makes a sound.`);
    }
}

class Dog extends Animal {
    makeSound(): void {
        console.log(`${this.name} barks: Woof!`);
    }
}

class Cat extends Animal {
    makeSound(): void {
        console.log(`${this.name} meows: Meow!`);
    }
}

// Polymorphic function - bisa menerima Animal apapun
function animalConcert(animals: Animal[]): void {
    for (let animal of animals) {
        animal.makeSound(); // Dynamic dispatch - panggil versi subclass
    }
}

let animals: Animal[] = [
    new Dog("Buddy"),
    new Cat("Whiskers"),
    new Animal("Generic")
];
animalConcert(animals);
// Output:
// "Buddy barks: Woof!"
// "Whiskers meows: Meow!"
// "Generic makes a sound."

2. Abstract Class

Abstract class adalah class yang tidak bisa di-instantiate. Digunakan sebagai blueprint untuk subclass. Bisa mengandung abstract methods (tanpa implementation) yang harus diimplementasikan oleh subclass.

// Abstract class - tidak bisa new Animal()
abstract class Animal {
    constructor(public name: string) {}

    // Abstract method - harus diimplementasikan oleh subclass
    abstract makeSound(): void;

    // Concrete method - bisa memiliki default implementation
    sleep(): void {
        console.log(`${this.name} is sleeping.`);
    }
}

// Dog harus implement makeSound
class Dog extends Animal {
    makeSound(): void {
        console.log(`${this.name}: Woof!`);
    }
}

// Cat juga
class Cat extends Animal {
    makeSound(): void {
        console.log(`${this.name}: Meow!`);
    }
}

// let a = new Animal(); // Error! abstract class
let dog = new Dog("Buddy");
dog.makeSound(); // "Buddy: Woof!"
dog.sleep();     // "Buddy is sleeping." (warisan)

3. Abstract Methods

Abstract method hanya deklarasi (tanpa body). Menjamin bahwa subclass akan memiliki implementasi tertentu.

abstract class Shape {
    constructor(public color: string) {}

    // Abstract method - MUST be implemented
    abstract area(): number;

    // Concrete method dengan call ke abstract
    describe(): string {
        return `Shape with color ${this.color}, area ${this.area()}`;
    }
}

class Circle extends Shape {
    constructor(color: string, public radius: number) {
        super(color);
    }

    // Implementasi abstract method
    area(): number {
        return Math.PI * this.radius * this.radius;
    }
}

class Rectangle extends Shape {
    constructor(color: string, public width: number, public height: number) {
        super(color);
    }

    area(): number {
        return this.width * this.height;
    }
}

let shapes: Shape[] = [
    new Circle("red", 5),
    new Rectangle("blue", 4, 6)
];
for (let s of shapes) {
    console.log(s.describe());
    // "Shape with color red, area 78.53..."
    // "Shape with color blue, area 24"
}

4. Polymorphism dengan Interface

Interface juga memungkinkan polymorphism tanpa inheritance.

interface Logger {
    log(message: string): void;
}

class ConsoleLogger implements Logger {
    log(message: string): void {
        console.log(`[Console] ${message}`);
    }
}

class FileLogger implements Logger {
    log(message: string): void {
        // write to file...
        console.log(`[File] ${message}`);
    }
}

class DatabaseLogger implements Logger {
    log(message: string): void {
        // insert into db...
        console.log(`[DB] ${message}`);
    }
}

// Function polymorphic - accepts any Logger
function processLogs(logger: Logger, messages: string[]): void {
    for (let msg of messages) {
        logger.log(msg);
    }
}

let logger: Logger = new ConsoleLogger();
processLogs(logger, ["Error 1", "Error 2"]);
// Bisa ganti dengan FileLogger atau DatabaseLogger tanpa ubah function

5. instanceof dan Type Guards

Saat bekerja dengan polymorphic references, terkadang perlu tahu tipe spesifik instance.

abstract class Animal {
    abstract makeSound(): void;
}

class Dog extends Animal {
    makeSound(): void { console.log("Woof"); }
    fetch(): void { console.log("Fetching ball"); }
}

class Cat extends Animal {
    makeSound(): void { console.log("Meow"); }
    climb(): void { console.log("Climbing tree"); }
}

function interact(animal: Animal): void {
    animal.makeSound(); // Polymorphic call

    // Type guard untuk cek specific type
    if (animal instanceof Dog) {
        animal.fetch(); // TypeScript tahu animal adalah Dog di sini
    } else if (animal instanceof Cat) {
        animal.climb();
    }
}

let a: Animal = new Dog("Buddy");
interact(a); // "Woof", "Fetching ball"

Latihan: Build Shape Hierarchy

abstract class Shape {
        constructor(public color: string = "black") {}

        // Abstract methods
        abstract area(): number;
        abstract perimeter(): number;

        // Concrete method
        describe(): string {
            return `${this.color} shape: area=${this.area().toFixed(2)}, perimeter=${this.perimeter().toFixed(2)}`;
        }
    }

    class Circle extends Shape {
        constructor(color: string, public radius: number) {
            super(color);
        }

        area(): number {
            return Math.PI * this.radius ** 2;
        }

        perimeter(): number {
            return 2 * Math.PI * this.radius;
        }
    }

    class Rectangle extends Shape {
        constructor(color: string, public width: number, public height: number) {
            super(color);
        }

        area(): number {
            return this.width * this.height;
        }

        perimeter(): number {
            return 2 * (this.width + this.height);
        }
    }

    class Triangle extends Shape {
        constructor(color: string, public a: number, public b: number, public c: number) {
            super(color);
        }

        area(): number {
            // Heron's formula
            let s = this.perimeter() / 2;
            return Math.sqrt(s * (s - this.a) * (s - this.b) * (s - this.c));
        }

        perimeter(): number {
            return this.a + this.b + this.c;
        }
    }

    // Polymorphic usage
    let shapes: Shape[] = [
        new Circle("red", 5),
        new Rectangle("blue", 4, 6),
        new Triangle("green", 3, 4, 5)
    ];

    for (let s of shapes) {
        console.log(s.describe());
    }

Rangkuman

  • Polymorphism: Satu interface, banyak implementasi
  • Subclass bisa override methods dari parent
  • Abstract class: tidak bisa di-instantiate, bisa punya abstract methods
  • Abstract method: deklarasi tanpa body, harus diimplementasikan subclass
  • override keyword bisa digunakan untuk explicit override (TypeScript 4.3+)
  • Polymorphism juga bisa via interface
  • Gunakan instanceof untuk type guards saat perlu tipe spesifik
