Interfaces: Contract untuk Tipe

Modul 4: Object-Oriented Programming

Kemajuan Belajar 0%

Apa itu Interface?

Interface adalah contract yang mendefinisikan struktur suatu object: properties, methods, dan tipe mereka. Class yang implements interface harus memenuhi kontrak tersebut.

Konsep: Interface mendefinisikan apa yang harus dimiliki sebuah type, bukan bagaimana implementasinya. Ini memisahkan contract dari implementation.

1. Interface Dasar

// Definisikan interface
interface User {
    id: number;
    nama: string;
    email: string;
    isActive?: boolean; // optional
}

// Class implements interface
class UserImpl implements User {
    id: number;
    nama: string;
    email: string;
    isActive: boolean = true; // optional tapi harus ada jika di-declare

    constructor(id: number, nama: string, email: string) {
        this.id = id;
        this.nama = nama;
        this.email = email;
    }
}

// Object literal juga bisa type-check terhadap interface
let user: User = {
    id: 1,
    nama: "Alice",
    email: "alice@example.com"
    // isActive optional, tidak wajib
};

2. Interface dengan Methods

Interface bisa mendefinisikan method signatures.

interface Animal {
    nama: string;
    makeSound(): void;
    move?(speed: number): void; // optional method
}

class Dog implements Animal {
    nama: string;
    breed: string;

    constructor(nama: string, breed: string) {
        this.nama = nama;
        this.breed = breed;
    }

    makeSound(): void {
        console.log(`${this.nama}: Woof!`);
    }

    // optional method bisa diimplementasi atau tidak
    move(speed: number): void {
        console.log(`${this.nama} runs at ${speed} km/h`);
    }
}

let dog: Animal = new Dog("Buddy", "Golden");
dog.makeSound(); // "Buddy: Woof!"
dog.move(10);    // "Buddy runs at 10 km/h"

3. Interface vs Type Alias

Keduanya bisa mendefiniskan object shape, tetapi ada perbedaan.

// Interface
interface Point {
    x: number;
    y: number;
}

// Type alias
type PointType = {
    x: number;
    y: number;
};

// Sebagian besar sama, tapi:
// - Interface bisa extends interface dan class (bukan type alias)
// - Type alias bisa union, intersection, mapped types
// - Interface lebih cocok untuk OOP, type alias untuk complex type manipulation

// Interface extends interface
interface NamedPoint extends Point {
    name: string;
}

// Type alias union
type ID = number | string;

Tips: Gunakan interface untuk public API contracts (class, object shape). Gunakan type untuk tipe kompleks (union, tuple, mapped).

4. Interface untuk Function

Interface bisa mendefiniskan function type juga.

// Interface untuk function signature
interface MathOperation {
    (a: number, b: number): number;
}

// Function yang sesuai dengan interface
let add: MathOperation = (a, b) => a + b;
let multiply: MathOperation = (a, b) => a * b;

console.log(add(2, 3));       // 5
console.log(multiply(2, 3));  // 6

// Class dengan method yang sesuai interface
class Calculator {
    operation: MathOperation;
    constructor(op: MathOperation) {
        this.operation = op;
    }
    calculate(a: number, b: number): number {
        return this.operation(a, b);
    }
}

let calc = new Calculator(add);
console.log(calc.calculate(5, 3)); // 8

5. Polymorphism via Interface

Interface memungkinkan polymorphism - berbagai class bisa diperlakukan sama jika implements interface yang sama.

interface Renderer {
    render(): void;
}

class Button implements Renderer {
    render(): void {
        console.log("Render button as HTML button");
    }
}

class Link implements Renderer {
    render(): void {
        console.log("Render link as HTML anchor");
    }
}

function renderAll(renderers: Renderer[]): void {
    for (let r of renderers) {
        r.render(); // polymorphic call
    }
}

let components: Renderer[] = [new Button(), new Link()];
renderAll(components);
// Output:
// "Render button as HTML button"
// "Render link as HTML anchor"

Latihan: Build Payment System

interface PaymentMethod {
    process(amount: number): Promise<boolean>;
    getName(): string;
}

class CreditCard implements PaymentMethod {
    constructor(public cardNumber: string, public expiry: string) {}

    async process(amount: number): Promise<boolean> {
        console.log(`Processing $${amount} with credit card ${this.cardNumber}`);
        return true; // simulate success
    }

    getName(): string {
        return "Credit Card";
    }
}

class PayPal implements PaymentMethod {
    constructor(public email: string) {}

    async process(amount: number): Promise<boolean> {
        console.log(`Processing $${amount} via PayPal (${this.email})`);
        return true;
    }

    getName(): string {
        return "PayPal";
    }
}

class PaymentProcessor {
    constructor(private methods: PaymentMethod[]) {}

    async pay(amount: number): Promise {
        for (let method of this.methods) {
            console.log(`Trying ${method.getName()}...`);
            let success = await method.process(amount);
            if (success) {
                console.log("Payment successful!");
                return;
            }
        }
        throw new Error("All payment methods failed");
    }
}

let processor = new PaymentProcessor([
    new CreditCard("1234-5678", "12/25"),
    new PayPal("user@example.com")
]);
processor.pay(100);

Rangkuman

  • Interface mendefinisikan kontrak (shape) untuk object atau class
  • Class menggunakan implements Interface untuk memenuhi kontrak
  • Properties dan methods harus sesuai dengan signatures di interface
  • Interface bisa extends interface lain
  • Bisa juga untuk function types
  • Interface enables polymorphism - berbagai class bisa diperlakukan sama
  • Interface vs type alias: interface lebih cocok untuk OOP contracts
