Encapsulation (Enkapsulasi)

Encapsulation adalah prinsip OOP yang menyembunyikan detail implementasi internal dan hanya mengekspos API yang diperlukan. Di TypeScript, kita mencapai ini melalui access modifiers: public, private, dan protected.

Goal: Lindungi data dari akses eksternal, kurangi coupling, dan tingkatkan maintainability.

1. public (Default)

Public members bisa diakses dari mana saja: di dalam class, subclass, dan dari instance eksternal.

class Person {
        constructor(public name: string, public age: number) {} // public by default

        greet(): void {
            console.log(`Hello, I'm ${this.name}`);
        }
    }

    let p = new Person("Alice", 25);
    console.log(p.name); // OK, public
    p.greet(); // OK, public

public adalah default - tidak perlu ditulis kecuali untuk kejelasan.

2. private

private members hanya bisa diakses di dalam class tersebut. Tidak bisa dari subclass atau instance lain.

class BankAccount {
        constructor(
            public accountNumber: string,
            private balance: number
        ) {}

        deposit(amount: number): void {
            this.balance += amount; // OK, dalam class
        }

        withdraw(amount: number): boolean {
            if (amount > this.balance) return false;
            this.balance -= amount;
            return true;
        }

        getBalance(): number {
            return this.balance; // OK, dalam class
        }
    }

    let acc = new BankAccount("12345", 1000);
    // console.log(acc.balance); // Error! private
    acc.deposit(500); // OK, public method
    console.log(acc.getBalance()); // 1500

Private untuk Data Hiding

Gunakan private untuk state internal yang tidak boleh diubah langsung dari luar. Berikan public methods (getter/setter) untuk controlled access.

3. protected

protected members bisa diakses di dalam class dan subclass-nya, tapi tidak dari instance eksternal.

class Animal {
        constructor(protected name: string, protected age: number) {}

        getInfo(): string {
            return `${this.name}, ${this.age} years`;
        }
    }

    class Dog extends Animal {
        breed: string;

        constructor(name: string, age: number, breed: string) {
            super(name, age);
            this.breed = breed;
        }

        // Bisa akses protected 'name' dan 'age' dari parent
        bark(): void {
            console.log(`${this.name} (${this.breed}) says woof! Age: ${this.age}`);
        }
    }

    let dog = new Dog("Buddy", 3, "Golden");
    dog.bark(); // OK
    // console.log(dog.name); // Error! protected, tidak bisa dari luar
    console.log(dog.getInfo()); // OK, public method

5. Parameter Properties Shortcut

Modifier di parameter constructor otomatis membuat property dengan modifier tersebut.

class Employee {
        // Parameter properties - otomatis buat property dengan modifier
        constructor(
            public id: number,
            private name: string,
            protected department: string
        ) {}

        getInfo(): string {
            return `ID: ${this.id}, Dept: ${this.department}`;
            // this.name accessible di sini (private tapi dalam class)
        }
    }

    let emp = new Employee(1, "Alice", "Engineering");
    console.log(emp.id); // OK, public
    // console.log(emp.name); // Error, private
    console.log(emp.getInfo()); // "ID: 1, Dept: Engineering"

6. Getter/Setter untuk Encapsulation

Gunakan getter/setter untuk memberikan controlled access ke private properties.

class Temperature {
        private _celsius: number = 0;

        get celsius(): number {
            // Bisa tambah validation atau computation
            return this._celsius;
        }

        set celsius(value: number) {
            if (value < -273.15) {
                throw new Error("Below absolute zero!");
            }
            this._celsius = value;
        }

        // Computed property (read-only)
        get fahrenheit(): number {
            return (this._celsius * 9/5) + 32;
        }
    }

    let temp = new Temperature();
    temp.celsius = 25; // panggil setter
    console.log(temp.celsius); // panggil getter: 25
    console.log(temp.fahrenheit); // 77 (computed)
    // temp.fahrenheit = 100; // Error! no setter

Latihan: Encapsulate Shopping Cart

class SecureShoppingCart {
        private items: { id: number; name: string; price: number; qty: number }[] = [];

        addItem(id: number, name: string, price: number, qty: number = 1): void {
            let existing = this.items.find(i => i.id === id);
            if (existing) {
                existing.qty += qty;
            } else {
                this.items.push({ id, name, price, qty });
            }
        }

        removeItem(id: number): void {
            this.items = this.items.filter(i => i.id !== id);
        }

        // Read-only computed properties
        get total(): number {
            return this.items.reduce((sum, item) => sum + (item.price * item.qty), 0);
        }

        get itemCount(): number {
            return this.items.reduce((count, item) => count + item.qty, 0);
        }

        // Public method untuk view contents
        getItems(): readonly { id: number; name: string; price: number; qty: number }[] {
            // Return copy to prevent external mutation
            return this.items.map(item => ({ ...item }));
        }
    }

    let cart = new SecureShoppingCart();
    cart.addItem(1, "Laptop", 1500000);
    console.log(cart.total); // 1500000 (via getter)
    console.log(cart.getItems()); // safe copy, original items protected

Rangkuman

  • public: accessible dari mana saja (default)
  • private: hanya dalam class tersebut
  • protected: dalam class dan subclass
  • Gunakan private untuk state internal, public untuk API
  • Protected untuk anggota yang perlu diakses subclass
  • Parameter properties shortcut: constructor(public x: T)
  • Kombinasikan dengan getter/setter untuk encapsulation yang kuat
