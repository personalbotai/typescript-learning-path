Object di TypeScript

Object adalah koleksi properti yang diidentifikasi oleh key (string atau symbol). TypeScript memberikan kontrol tipe yang ketat untuk properti object.

1. Object Literal (Inline)

Cara paling sederhana membuat object langsung dengan curly braces.

// Object dengan properti string dan number
let user = {
    nama: "Alice",
    umur: 25,
    aktif: true
};

// Akses properti
console.log(user.nama); // "Alice"
console.log(user["umur"]); // 25 (bracket notation)

// TypeScript inferred: { nama: string; umur: number; aktif: boolean; }

// Modifikasi properti
user.umur = 26;
user.email = "alice@example.com"; // Error! 'email' tidak ada di tipe inferred

Note: TypeScript membuat tipe otomatis dari object literal. Setelah dibuat, tidak bisa menambahkan properti baru tanpa error.

2. Interface untuk Object Tipe

Gunakan interface untuk mendefinisikan kontrak object yang bisa digunakan di banyak tempat.

// Definisikan interface
interface User {
    nama: string;
    umur: number;
    email?: string; // optional property
}

// Gunakan interface untuk type annotation
let user1: User = {
    nama: "Bob",
    umur: 30,
    email: "bob@example.com"
};

let user2: User = {
    nama: "Charlie",
    umur: 25
    // email opsional, tidak wajib
};

// TypeScript akan memeriksa:
// - semua required properties ada
// - tipe setiap properti sesuai
// - tidak ada properti ekstra (strict mode)

// Function yang menerima User
function greet(user: User): string {
    return `Halo, ${user.nama}!`;
}

3. Type Alias untuk Object

Selain interface, bisa pakai type untuk membuat tipe object.

// Type alias untuk object
type Product = {
    id: number;
    name: string;
    price: number;
    inStock: boolean;
};

// Gunakan seperti interface
let p1: Product = {
    id: 1,
    name: "Laptop",
    price: 1500000,
    inStock: true
};

// Interface vs Type Alias:
// - Interface bisa di-extends, type alias bisa union/intersection
// - Keduanya bisa untuk object, pilih sesuai preferensi

4. Optional, Readonly, dan Index Signatures

TypeScript memberikan modifiers untuk menyesuaikan object types.

// Optional property (?)
interface Config {
    host: string;
    port: number;
    ssl?: boolean; // optional, bisa tidak ada
}

// Readonly property
interface Point {
    readonly x: number;
    readonly y: number;
}

let p: Point = { x: 10, y: 20 };
p.x = 30; // Error! x adalah readonly

// Index signature (dynamic keys)
interface Dictionary {
    [key: string]: number; // semua properti string â†’ number
}

let dict: Dictionary = {
    satu: 1,
    dua: 2,
    tiga: 3
};

5. Type Inference untuk Object

TypeScript bisa menebak tipe object literal tanpa explicit annotation.

// TypeScript inferred type dari object literal
let car = {
    brand: "Toyota",
    year: 2020,
    isElectric: false
};
// Inferred type: { brand: string; year: number; isElectric: boolean; }

// Bisa explicit jika ingin
let car2: { brand: string; year: number } = {
    brand: "Honda",
    year: 2022
};

6. Object Methods

Function bisa menjadi properti object.

interface Calculator {
    tambah(a: number, b: number): number;
    kurangi(a: number, b: number): number;
}

let calc: Calculator = {
    tambah: (a, b) => a + b,
    kurangi: (a, b) => a - b
};

console.log(calc.tambah(5, 3)); // 8
console.log(calc.kurangi(5, 3)); // 2

Latihan: Modelling Data Mahasiswa

Buat interface untuk mahasiswa dan function untuk menghitung IPK.

interface Mahasiswa {
    nama: string;
    nim: string;
    nilai: number[]; // array of grades (0-100)
}

function hitungIPK(m: Mahasiswa): number {
    if (m.nilai.length === 0) return 0;
    let sum = 0;
    for (let n of m.nilai) {
        sum += n;
    }
    return sum / m.nilai.length;
}

let andi: Mahasiswa = {
    nama: "Andi",
    nim: "123456",
    nilai: [80, 85, 90, 78]
};

console.log(`IPK ${andi.nama}: ${hitungIPK(andi).toFixed(2)}`);
// IPK Andi: 83.25

Kesalahan Umum

Property missing

interface User { name: string; age: number; }
let u: User = { name: "Ali" }; // Error: Property 'age' missing

Extra property

interface User { name: string; }
let u: User = { name: "Ali", extra: 123 }; // Error: 'extra' not in interface

Rangkuman

  • Object literal: { key: value, ... }
  • Gunakan interface atau type untuk mendefinisikan tipe object yang reusable
  • Optional properties: prop?: type
  • Readonly properties: readonly prop: type
  • Index signatures untuk dynamic keys: [key: string]: type
  • TypeScript melakukan strict checking pada object shape
