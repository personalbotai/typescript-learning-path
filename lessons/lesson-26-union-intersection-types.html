Union dan Intersection Types

TypeScript menyediakan dua operator powerful untuk menggabungkan tipe: | (Union) dan & (Intersection). Mengerti kedua konsep ini esensial untuk membuat tipe yang fleksibel namun tetap type-safe.

1 Union Types (|)

Union type memungkinkan sebuah nilai memiliki salah satu dari beberapa tipe yang ditentukan. Gunakan operator | untuk memisahkan tipe-tipe tersebut.

function printId(id: number | string) { if (typeof id === "string") { console.log("ID sebagai string:", id.toUpperCase()); } else { console.log("ID sebagai number:", id.toFixed(2)); } } printId(101); // OK printId("202"); // OK // printId(true); // Error: Argumen bertipe 'boolean' tidak dapat ditetapkan ke parameter bertipe 'number | string'

‚ö†Ô∏è Penting: Ketika menggunakan union, TypeScript hanya mengizinkan operasi yang valid untuk semua anggota union. Untuk mengakses properti/metode spesifik, gunakan type guards (akan dibahas di Lesson 27).

2 Intersection Types (&)

Intersection type menggabungkan beberapa tipe menjadi satu tipe yang memiliki semua properti dari setiap tipe yang digabung. Gunakan operator &.

interface User { name: string; age: number; } interface Contact { email: string; phone: string; } // Gabungkan User dan Contact menjadi Admin type Admin = User & Contact; const superAdmin: Admin = { name: "Budi", age: 35, email: "budi@company.com", phone: "+628123456789" // ^? Admin memiliki semua properti dari User dan Contact };

Intersection sangat berguna untuk mixing tipe, misalnya menggabungkan interface dengan type, atau menggabungkan beberapa interface menjadi tipe baru yang lebih kaya.

3 Use Cases Praktis

a. Event Handlers

type MouseEvent = { x: number; y: number }; type KeyboardEvent = { key: string; ctrlKey: boolean }; function handleEvent(e: MouseEvent | KeyboardEvent) { if ("x" in e) { console.log(`Mouse di (${e.x}, ${e.y})`); } else { console.log(`Tombol: ${e.key}`); } }

b. Config Objects

interface ServerConfig { host: string; port: number; } interface DatabaseConfig { dbName: string; user: string; password: string; } type AppConfig = ServerConfig & DatabaseConfig; const config: AppConfig = { host: "localhost", port: 5432, dbName: "mydb", user: "admin", password: "secret" };

4 Common Pitfalls

‚ùå Union tidak bisa mengakses properti spesifik tanpa type guard

// id: string | number
// id.length // Error! 'length' hanya ada di string

‚ùå Intersection mungkin bertabrakan jika properti namanya sama

interface A { x: number; }
interface B { x: string; }
type C = A & B; // x: never (tidak mungkin) - bertabrakan!
